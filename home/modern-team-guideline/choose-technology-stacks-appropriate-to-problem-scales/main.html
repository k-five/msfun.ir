choose-technology-stacks-appropriate-to-problem-scales

Nowadays that we have variety of technologies - specially open source - it makes hard to choose the right one.

The right choice is not to choose the best in terms of fully featured one, but the one which is best for our needs.

The size of the problem we have, the complexity of the solution we need, the amount of time we want to use a technology, are some keys to choose an appropriate technologies.

We should not choose a technology because we feel good about, we think it is cool, etc. If we can solve it with a simple one, we use that simple one.

different services

If we have different services - which usually they are - then we should not think of the same database for all services we have. If we can pass over it by using a simple JSON file we will do that.

If for a service we need mongoDB then we do not use JSON and will choose mongoDB, of course.

So the scale of our problems, or needs determines which technologies we should choose..
