Code and deliver it to me!

The old way of developing software in most organizations is that someone comes in, starts to code and delivers it to someone else. This "someone else" can be PM, or CTO. etc.

Since in this model the developers are not given the responsibilities of their works; they just code and deliver it to someone else. A common pattern mostly in Monolithic Architecture we can see and still exists.

Usually in this model the organization does not have pipeline or any kind of automation operation and things are done manually. Also they do not push new code and features to production frequently and because of this huge huge amount of code should be pushed at once.

And big chuck of code deployment almost always has the risk of breaking the old code / service or whatever we have.

You build it, you run it

Amazon started to change their monolithic architecture at about 2000 towards microservice architecture.

In about 2004 Werner Vogels joined Amazon.

His famous quote you build it, you run it is the way of describing a software developer responsibility.

In this way, developers do not just code and sit around. They test, deploy, etc and in fact they take care of the pipeline of their codes.

How mush time does it take to deploy the code? Very little. Everything has been automated and deployment equals just pushing to the main repository or merging to master branch.

How many days does it take to push to production code? Zero. They can push to production everyday or even every hour if they want.

Good team, Good organization

A good team or good organization do not grab code from their developers and then push it to production.

They have pipeline and automation. Developers code and after they commit their code, using Version Control System, an automated system helps them to push the code they wrote to production.

Testing has been automated. Deployment has been automated. Building has been automated. Monitoring has been automated.

So they code, test, deploy, build, and monitor over and over again. No code delivers to PM, CTO, etc and this is the right our team, developers, organization should follow.
