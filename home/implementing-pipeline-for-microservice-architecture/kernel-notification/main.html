kernel-notification

We have to main pattern for automating a task using a computer.

  1. Polling
  2. Watching

Polling

This is polling:

 while tree; do
	sleep 1;
	# checking something by continuously doing a command
	# for example cron-tab in Linux
 done;

Polling is everywhere. If you call your friends many times continuously to see where he / she is, this is poling.

This way of automating thins is the old way, but for some cases we still need it.

Also it is synchronous which means we know when it is going to happen.

When we have repeated tasks that happens periodically we can use polling to automate them. Like what? Like backuping a database we have every 24 hours.

We know we have users they daily do CRUD: Create, Read, Update, Delete so we automate our backup system for safety.

One drawback of polling is that it is not good when we have asynchronous change and we want to do something based on that changes.

For example we want to be aware if file X has been changed or has been deleted. Using polling is possible like so:

 while tree; do
	sleep 1;
	if [ -e file-X ]; then
		# do something
	fi
 done;

But what if the file has not been deleted in an hour, or day, or even month! Should we run 30 days x 24 hours x 60 minutes x 60 seconds if statement just for this check?

Definitally it is inefficient

Watching

I do not know much about Window Kernel but a little about Linux Kernel so I use it in this context but I think the Window Kernel should somehow have the same features.

Based on this fact to polling is inefficient to observe active in our Operating System specially Hard Disk after version 2.7 in Linux Kernel an API was added to asynchronous polling which the right term for it is watching.

This API is inotify(7).

When we open a file, who opens the file for us?

When we open a directory, who?

When we write to a file or create a directory, who does the task?

The answer to all is: The Kernel.

The kernel knows when a file opened because it is done by itself. So the kernel is aware of all the actives happening with the Hard Disk.

If we had a way to register our need - like modifying a file - with the kernel, then whenever that need happens the kernel can notify us.

Sure the open source communities are smart and done this for us which is inotify.

Using this APi we can be notified whenever a change happens to any kind of activities with our File-System. See man 7 inotify for more.

The kernel can do with Hard Disk what the browsers (= FireFox, etc) does with DOM. Notifying event to the users.
