what-is-a-pipeline-for-microservices

While in Monolithic Architecture we have a little or not so much operations, in Microservices we have many.

For example we have:

  • Source Code
  • Some kind of Tests
  • Maybe Building Locally
  • Shared Repository
  • CI Server
  • Production Serer

Then multiply these by number of services we may have, for example 10.

Plus we usually have team-pre-services (= Cross Functional Team), which means a team is responsible for a service, then we have many repeatable operations for our products.

If we somehow do not create / find a pipeline for these operations then we will have headache + high-waisted time, energy, etc.

Thus our goal is implement a road with which our source code can be delivered to our end-user.

And these steps (how many we have) from source code to end users should be done effectively which in really means in a automated way.

So our pipeline simply means: multi-steps that have been automated from our source code to deliver our product to our end-user.
