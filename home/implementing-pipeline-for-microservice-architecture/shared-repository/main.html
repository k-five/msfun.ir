shared-repository

Why do not we just push the built code directly to the production server?

For example simply using rsync via SSH we can send it.

The reason is simple: we want that others can also code / interact with our repository.

If we just keep it / them offline on our local machine, no one can use it / them. Yes some developers in the same company can, but how about outside of the company?

Shared repository not only helps doing that, but also can been seen as a backup for our codes.

So based on previous step which was continuous integration, we add another step to our process which is a shared repository; it can be one or more than one depending upon our needs.

So if you have code under Version Control then put it on GitHub or somewhere else to have a shared repository.

We use the Web-Hook of this shared repository to notify our CI Server so our code on the server can be synchronized with GitHub repository.
